# RePERFORMANCE 시스템 구조

## 애플리케이션 책임

하나의 Next.js 애플리케이션이 공개 홈페이지와 상담 전 운영 화면을 제공합니다.

| 영역 | 기준 위치 | 책임 |
| --- | --- | --- |
| 공개 홈페이지 | `app/`, `app/services/` | 브랜드·서비스 안내, 상담 전환 |
| 체대입시 정보 | `app/pe-exam/` | 대학·전형 정보, 상담 준비형 AI |
| 상담 신청 | `app/apply/` | 최소 필수 정보 수집, 유입·예약 정보 연결 |
| 관리자 운영 | `app/admin/`, `components/rp-consultation/` | 고객 상태, 상담, 방문 가능 시간, AI 승인 |
| API | `app/api/` | 인증·권한·입력 제한을 거친 서버 기능 |
| 도메인 로직 | `lib/` | 데이터베이스, 세션, 보안, 전환, 보관 정책 |
| 운영 검증 | `scripts/` | 배포 전·후 자동 점검 |
| 외부 어댑터 | `integrations/` | 선택적 외부 서비스의 단일 기준 소스 |

관리자 상담 화면은 `components/rp-consultation/RPConsultationMode.jsx` 한 파일을 기준으로 사용합니다. 일반 상담과 체대입시 상담을 같은 흐름에서 처리하며 별도 복사본을 두지 않습니다.

## 데이터 기준

| 데이터 | 최종 관리 기준 |
| --- | --- |
| 상담 신청, 계정, 상담 전 고객 상태, 방문 상태 | RePERFORMANCE PostgreSQL |
| 계약서, 개인정보 동의서, 결제·환불 원본 | Google Workspace |
| 상담 방문·첫 수업·실제 수업 일정 | Calendar |
| 계약 완료 후 운동·회원관리 | NORE |

Google Drive/Sheets는 기본 저장소가 아닙니다. 운영자가 명시적으로 활성화한 경우에만 제한된 백업 또는 전환기 fallback으로 사용합니다. NORE는 안내 문구로만 연결하며 API, webhook, 자동 전송을 추가하지 않습니다.

## 코드 통합 원칙

1. 공개·관리자 화면은 같은 `app/` 프로젝트 안에서 라우트로 구분합니다.
2. 인증과 권한 확인은 `lib/rpSessionAuth.js`와 서버 API에서 수행합니다.
3. 데이터베이스 규칙은 `lib/rpDatabase.js`와 날짜순 마이그레이션을 함께 갱신합니다.
4. 브라우저 컴포넌트는 `rpClientApi.js`를 통해 내부 `/api/rp/*`만 호출합니다.
5. 외부 서비스별 프로젝트 복사본을 만들지 않고 선택적 어댑터와 환경변수로 분리합니다.
6. 생성 데이터와 정적 자산은 각각 `app/pe-exam/`과 `public/`의 단일 기준 파일만 유지합니다.
